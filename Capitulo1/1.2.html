Análisis de los datos. Etapas

Un modelo necesita de datos. Se divide en variables y patrones.
Una variable es una característica. Un patrón es un conjunto de variables.
Un patrón puede ser una imagen, y una variable sería cada píxel.

Etapas de la producción

Preprocesado de datos

Es la parte que tarda más tiempo. Consiste en las siguientes partes:

  • Limpieza de datos: Se determina si hay datos atípicos o incorrectos.
    Un dato atípico es algo improbable, pero no imposible. Un dato incorrecto sí es imposible
  • Normalización: Se busca que el rango de los datos de entrada no sea tan grande.
    Los datos no deben distar tanto entre sí
  • Detección de outliers: Un outlier es un dato atípico. Disminuyen la eficacia del modelo.
    Para saber si existen atípicos, se compara mediana y media. Si hay una gran diferencia, hay atípicos.
    También se puede ver si los valores están dentro del rango (m-3σ, m+3σ], donde m es la media y σ la desviación estándar.
    Para detectarlos, se usan algoritmos de agrupamiento de datos y el isolation forest
  • Datos ausentes: Para tratar con datos ausentes, se tiene una estrategia enfocada en variables y otra enfocada en patrones.
    Si se enfoca en variables, se reemplaza lo faltante con la media o la moda, si la variable es continua o discreta, respectivamente.
    Si se enfoca en patrones, se reemplaza lo faltante con datos de un patrón similar. Se usan distintos algoritmos
  • Ingeniería de características: Se transforman, seleccionan y analizan las entradas para darle "masticado" al modelo.
    Se define la información que se le va a pasar al modelo

Análisis exploratorio de datos

Se describen los datos con herramientas estadísticas.
Se determina datos estadísticos, relaciones lineales, normalidad, independencia y estadística inferencial de los datos obtenidos.
Se aplican algoritmos de agrupamiento (clustering) para agrupar los datos y simplificar problemas.

Desarrollo del modelo

Se puede utilizar Machine Learning (ML) o Deep Learning (DL) para desarrollar el modelo.
Se toma en cuenta el No free lunch theorem: no existe un mejor modelo para todos los problemas. Es necesario buscar el mejor modelo para el problema.
Sobreajuste: Es cuando el modelo no es capaz de generalizar. Se ajusta bien a los datos de entrenamiento, pero no da buenos datos de salida, como que se queda bloqueado en los datos de entrenamiento.
Para evitar esto, se dividen los datos en tres partes:

  1. Datos de entrenamiento: Datos para ajustar el modelo
  2. Datos de validación: Controla el ajuste del conjunto de entrenamiento
  3. Datos de test: Define el comportamiento del modelo ante datos no observados
Con estos datos se determina la eficacia del modelo para los datos de entrenamiento y de validación. El error en ambos va bajando, pero cuando aumenta el error de validación implica que se sobreajusta.
Se debe parar cuando esto pase, ya que el modelo tiene más capacidad de generalización. Oviamente estos datos deben estimarse ya que no se ven tan fácilmente.
También se puede utilizar un k-fold. Se usa cuando hay pocos patrones. Se dividen los datos en conjuntos, y luego se agrupan en conjuntos de subconjuntos (folds).
Se usa uno de esos subconjuntos para validar y el resto para entrenar. Se divide en k valores para el conjunto de validación.
Las divisiones del fold se hacen automáticamente por el lenguaje.

Análisis de errores

Se debe tomar en cuenta: la medida de error que dirige el desarrollo de modelos, y la medida del error que se quiere analizar. En todo caso, se evalúa el sesgo y varianza.

  • Sesgo: Se define como la cantidad que mide la tendencia a aprender de forma sistémica cosas equivocadas.
    El sesgo deriva al modelo a un tipo determinado de resultado
  • Varianza: Se define como la tendencia a aprender cosas irrelevante para lo que se quiere aprender.
    El sesgo dda respuestas demasiado específicas para el problema en concreto
El objetivo es reducir el sesgo y varianza lo más posible.

Producción

Se debe tener en cuanta la evolución del modelo y lo que se quiere que haga.
Para desarrollar un modelo de estos se utiliza la metodología CRISP-DM. Tiene tres partes y tres ppreguntas clave:

  • ¿Qué se quiere resolver?
  • ¿Qué mdeida se va a usar para comprobar el funcionamiento del modelo?
  • ¿Qué datos se usarán para comprobar la solución del problema?
Se busca modularizar los proyectos y no hacerlos demasiado grandes. También se busca la comunicación entre el analista de datos y el cliente.
